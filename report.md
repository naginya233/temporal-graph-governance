# 整合包阶段汇报小文档（核心思路与创新点）

## 1. 项目定位与目标
本整合包将两个子系统统一到同一工程中：

- 交通治理分析引擎：`traffic_agent_system`
- 治理主控与可视化审阅端：`DairV2X_SceneGraph_Validator`

总体目标是构建一个“可运行、可审阅、可实验”的闭环系统，在不改动视觉到场景图生成链路的前提下，充分利用场景图开展交通治理分析与事件级研判。

## 2. 核心思路
核心思路可以概括为三句话：

- 用关系拓扑理解交通行为：将场景图中的车-车关系转成治理语义（冲突、让行、跟驰、死锁等）。
- 用时序一致性抑制帧级抖动：避免单帧误判导致风险等级频繁跳变。
- 用前端主控打通工程闭环：前端可控后端运行，支持运行管理、日志追踪、风险审阅与结果回看。

## 3. 系统流程（端到端）
1. 输入对齐：按帧对齐 Scene Graph JSON、BEV 图、原图。
2. 关系治理分析：识别让行失序、冲突传播链、潜在死锁、跟驰瓶颈等结构性风险。
3. 轻量化剪枝：在不破坏关键关系的前提下压缩关系拓扑，降低推理负担。
4. 帧级事件推理：规则驱动风险评分，可选本地 LLM 做补充语义解释。
5. 时序一致性校准：对风险做时间维度平滑与稳定化，输出校准后风险。
6. 事件分段：将帧级风险聚合为连续事件段，形成可解释的时序事件单元。
7. 结果输出：生成 JSONL、summary、Markdown/HTML 报告。
8. 前端审阅：在主控台完成运行控制、状态查看、日志监控与风险帧审核。

## 4. 主要创新点

### 创新点 1：时序一致性校准模块（已落地）
针对“帧级风险易抖动”的实际问题，引入时序一致性校准：

- 持续风险增强：连续出现的高风险关系边给予稳定增益。
- 瞬态噪声抑制：仅单帧出现的可疑关系边给予惩罚。
- 指数滑动平滑：通过 EMA 平滑风险分数并支持空风险衰减。

效果上可显著减少风险等级的无效波动，提高事件段边界的稳定性与可解释性。

### 创新点 2：治理主控优先的双模式审阅
将原关系校对工具升级为“治理主功能 + 校对次功能”的统一主控台：

- 前端可直接启动/停止治理管道。
- 后端异步运行，支持实时日志与运行状态追踪。
- 运行完成后可无缝进入风险审阅闭环。

### 创新点 3：研究执行层（Research Ops）内置
内置实验协议、消融矩阵、批跑脚本与结果汇总脚本，使研究验证从“手工临时操作”升级为“可复现实验流程”。
